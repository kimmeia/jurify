SmartFlow — Visual repaginado dos nós

📐 Antes vs Depois 🎨 Galeria ⚡ Estados 💡 Tooltips

Sobre este mockup

Visual proposto pros nós do canvas. Header colorido por categoria (em vez de chip pálido), preview da configuração dentro do nó (em vez de "3 categorias" genérico), e chips das variáveis que ele publica (clica e copia {`{{var}}`}). Status dot + validação inline pra mostrar problemas de config sem precisar abrir o painel. Zero alteração no motor — só fachada visual.

Hover em qualquer label do painel direito mostra a explicação completa com exemplo. Não precisa decorar nem abrir docs.

Antes (sem tooltip)

📦 Resumo do PR-7

Componente

PassoNodeView repaginado

Header gradient por categoria + body com preview da config (chave/valor formatado) + footer com chips das variáveis publicadas (clica e copia {`{{var}}`}) + status dot + validação inline.

UX

Tooltips + validação visível

Cada label do painel direito ganha tooltip com exemplo. Validação que hoje só aparece no painel também vira chip vermelho/âmbar no próprio nó — usuário vê problema sem clicar.

Onboarding

Wizard "primeiro fluxo"

Cenário novo abre num modal de 3 passos guiados em vez de canvas em branco. Cria um cenário pré-configurado que o usuário só edita.

Zero alteração no motor

Tudo é frontend — função pura resumirConfigParaNo(tipo, config) que devolve as linhas de preview, mesma estrutura que o painel direito usa pra renderizar. Os 2085 testes do backend continuam intocados.
